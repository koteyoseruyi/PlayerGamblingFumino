# PlayerGamblingFumino

一个玩家对赌插件，全程零和，无庄家抽水。

## 功能简介

- **玩家对赌**：通过猜拳游戏进行赌局

## 依赖

| 依赖 | 说明 |
|------|------|
| Paper 1.21+ | 服务端核心 |
| Vault | 经济系统接口（需搭配经济插件如 EssentialsX、CMI 经济等使用） |

## 指令

| 指令 | 权限 | 描述 |
|------|------|------|
| `/gamble invite <玩家>` | 无 | 向一名玩家发起对赌邀请 |
| `/gamble accept` | 无 | 接受收到的对赌邀请 |
| `/gamble disaccept` | 无 | 拒绝收到的对赌邀请 |
| `/gamble reload` | gambling.admin | 重载配置文件 |

## 配置文件说明

```yaml
settings:
  # 基础步长（第一回合变动幅度）
  base-step: 0.2

  # 每回合步长增量（越往后步长越大）
  gradient-step: 0.04

  # 邀请超时时间（秒）
  invite-timeout: 60

  # 每轮结果展示时间（秒）
  result-display-time: 5
```

## 🎲 游戏规则

### 1. 开局设定

双方通过 GUI 选择以下内容，**双方必须选择一致** 才能确认开始：

| 设定项 | 选项 |
|--------|------|
| **回合数** | 5 / 7 / 9 回合 |
| **金额档位** | 10,000 / 25,000 / 50,000 / 250,000 / 500,000 |

确认后，系统从双方 Vault 账户各扣除 **档位金额**，总奖池 = **2 × 档位金额**。

---

### 2. 天平拔河

- **甲方** 初始系数为 **1.0**，**乙方** 系数 = `2.0 - 甲方系数`
- 每轮双方通过 **石头剪刀布** 决定系数走向

### 3. 递增步长

第 k 回合的步长 = `baseStep + (k - 1) × gradientStep`

例：

| 回合 | 步长 | 回合 | 步长 |
|:---:|:---:|:---:|:---:|
| 1 | 0.20 | 6 | 0.40 |
| 2 | 0.24 | 7 | 0.44 |
| 3 | 0.28 | 8 | 0.48 |
| 4 | 0.32 | 9 | 0.52 |
| 5 | 0.36 | | |

- **甲方赢** → 甲方系数 **+ 步长**
- **乙方赢** → 甲方系数 **− 步长**
- **平局** → 系数不变，**步长累积到下一回合**

### 4. 触边即终结

- 甲方系数 **≥ 2.0** → **甲方全胜**，获得全部奖池
- 甲方系数 **≤ 0.0** → **乙方全胜**，获得全部奖池
- 达到预设回合数 → 按最终系数比例分配

### 5. 奖金分配

```
甲方获得 = 奖池 × (甲方系数 / 2)
乙方获得 = 奖池 × (1 − 甲方系数 / 2)
```

- 系数 **1.0** → 各得 50%（回到起点）
- 系数 **1.5** → 甲方 75%，乙方 25%
- 系数 **0.5** → 甲方 25%，乙方 75%
- 系数 **2.0** → 甲方 100%（触顶全胜）
- 系数 **0.0** → 乙方 100%（触底全胜）

---

## ⚠️ 注意事项

- 游戏期间 **无法关闭 GUI**，通过掉线强行退出视为 **自动判负**
- 退出方押注归胜者所有
- 插件依赖 Vault 经济系统，未安装时无法开始游戏
